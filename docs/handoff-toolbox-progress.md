## 当前目标

Vistack 的主生图工作台已经比较完整，本阶段的重点是新增一个不破坏主流程的「创作工具箱」。工具箱用于承载图片反推提示词、自定义模特、一键换装、遮罩编辑等辅助能力，并且让部分工具可以直接在工具箱内完成生成，减少来回切换。

核心原则：

- 主工作台的生图、历史、资产库、API 配置尽量不重排、不削弱。
- 工具箱复用现有生图 API、队列、历史保存能力，不另起一套生成系统。
- 工具箱可以独立显示自己的生成结果，避免把主工作台最近结果混进来。
- 用户可见文案避免再使用「草稿」概念，统一为「可编辑提示词」。

## 已完成内容

顶部视图切换从「工作台 / 资产库」扩展为「工作台 / 资产库 / 工具箱」。

相关文件：

### 2. 图片反推提示词

工具箱内新增图片反推提示词模式：

- 可上传多张图片。
- 使用提示词助手 API 读取图片。
- 输出适合生图模型复用的结构化提示词。
- 反推结果可以加入右侧「可编辑提示词」。

注意：图片反推本身不直接生图，它是提示词辅助工具。用户可以把反推结果加入右侧提示词后，配合其他工具参考图直接生成。

相关文件：

- `src/components/ToolboxPanel.vue`
- `src/services/api.ts`
- `src/types/index.ts`

### 3. 自定义模特

工具箱内新增自定义模特模式：

- 上传人物参考图。
- 支持选择模特资产类型：
  - 脸部角度包：正面、45 度、侧面，强调面部一致性。
  - 全身三视图：正面、侧面、背面，强调身材比例和穿搭调用。
  - 头像/胸像表情包：面部、发型、神态稳定。
  - 姿态参考包：站姿、坐姿、走姿、半身动作。
- 支持选择用途：
  - 通用调用、换装、头像写真、商业图、双人合影、连续剧情。
- 支持身份保真强度：
  - balanced / high / strict。
- 可保存为本地模特资产。
- 可直接在工具箱内生成。
- 可送回主创作台继续精修。

自定义模特资产保存位置：

- IndexedDB 数据库：`vistack-toolbox`
- 工具文件：`src/utils/toolboxAssetsDb.ts`

### 4. 一键换装

工具箱内新增一键换装模式：

- 上传人物参考图。
- 上传服装参考图，或填写服装描述。
- 可选择服装参考优先级：
  - 自动判断
  - 服装图优先
  - 文字描述优先
  - 融合图文
- 可选择保留项：
  - 身份、发型、姿态、身材、背景、光线。
- 可选择背景处理：
  - 保留原背景、棚拍背景、街拍环境。
- 可选择输出风格：
  - 真实照片、商业广告、lookbook。
- 可直接在工具箱内生成。
- 可送回主创作台继续精修。

### 5. 遮罩编辑

工具箱内新增遮罩编辑模式：

- 上传底图。
- 在本地画布上涂抹白色遮罩。
- 支持画笔大小、橡皮、清空、导出黑白遮罩。
- 可填写局部修改要求。
- 生成时会把底图和遮罩作为参考图一起提交。
- 当前实现将 mask 作为视觉参考约束传给已有生图接口，不是原生 multipart mask/edit 接口。

设计说明：

- 这样实现最快、最稳定，也能兼容 nano banana、grsai、sub2api 这类图生图代理。
- 如果后续要接原生编辑接口，需要在 `buildGenerateRequest` 和 provider 分支里单独支持 mask 字段。

### 6. 工具箱直接生成

自定义模特、一键换装、遮罩编辑均已支持工具箱内直接生成。

行为：

- 不需要先点「生成提示词」。
- 当资料足够时，点击「直接生成」会自动组装提示词。
- 右侧「可编辑提示词」支持用户单次微调后点击「直接生成」。
- 微调后生成会自动使用当前工具页的参考图：
  - 自定义模特：人物参考图。
  - 一键换装：人物参考图 + 服装参考图。
  - 遮罩编辑：底图 + 遮罩图。

相关文件：

- `src/components/ToolboxPanel.vue`
- `src/App.vue`

### 7. 工具箱独立结果区

工具箱内有独立的结果显示区域和工具箱任务队列。

设计：

- 工具箱任务会被标记为 `origin: 'toolbox'`。
- 工具箱结果显示只读取工具箱任务。
- 主工作台最新结果不会混入工具箱结果区。
- 工具箱任务仍然复用现有历史保存逻辑，所以生成成功后会进入资产库/历史。
- 工具箱任务的 running 状态不会让主工作台「参考图生成」按钮误显示生成中。

相关类型：

```ts
export interface GenerationTask {
    source: 'text' | 'image'
    origin?: 'studio' | 'toolbox'
}
```

相关文件：

- `src/App.vue`
- `src/components/ResultDisplay.vue`
- `src/types/index.ts`

## 继续开发建议

### 优先级 1：实际试用工具箱直接生成

重点看：

- 自定义模特直接生成是否符合预期。
- 一键换装自动组装提示词是否足够清楚。
- 遮罩编辑作为参考图约束是否能被当前模型理解。
- 工具箱结果区是否只显示工具箱任务。
- 工具箱任务是否进入历史/资产库。

### 优先级 2：工具箱 UI 细节

目前工具箱能用，但仍可继续打磨：

- 右侧「可编辑提示词」按钮数量偏多，可以进一步整理成主按钮 + 次级操作。
- 工具箱结果区可能需要更紧凑的瀑布流视觉。
- 遮罩画布的笔刷光标、缩放、撤销可以后续增强。
- 参考图预览和删除交互可以统一得更精致。

### 优先级 3：导入导出

因为自定义模特资产、历史、模板、API 预设都在浏览器本地，换机器时不会自动迁移。

后续可做：

- 导出全部本地设置为 JSON。
- 导入 JSON 恢复：
  - API 预设
  - 自定义词组
  - 自定义模板
  - 自定义模特资产
  - 资产库分类

### 优先级 4：原生 mask/edit 接口

当前遮罩是参考图约束，兼容性好，但不是原生编辑接口。

如果未来某些 provider 支持原生 mask：

- 扩展 `GenerateRequest`。
- 在 provider 构造 payload 时加入 mask 字段。
- UI 上标识「视觉遮罩」和「原生遮罩」两种模式。

## 注意事项

- 不要为了统一把生图 API 和提示词助手 API 合并，用户已经明确不喜欢这个改动。
- 不要把主工作台改成纯画布模式，用户希望新增模式，而不是替换现有方便的生图区。
- UI 调整要优先清晰易用，不要为了压缩空间牺牲一眼可懂。
- API 高级配置先保持稳定快速实现，不要过度复杂化。
- 工具箱应该是辅助功能集合，不只是提示词工具。
